## 🎯 DOCUMENTATION PHILOSOPHY

**What We Keep:**
- Standards we follow
- System architecture we implement
- Business rules we enforce
- Operational procedures we execute
- Technical specifications we build to

**What We Don't Keep:**
- Historical "fix" documents
- Session summaries of changes
- Investigation reports
- Audit reports from past issues
- "Complete" status documents

**Why:**
- The system IS the documentation
- Code reflects current state
- Standards guide future work
- No need to document every change
- Focus on "what to follow" not "what we fixed"
